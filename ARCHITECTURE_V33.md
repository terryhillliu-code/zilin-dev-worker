# 知微系统 (Zhiwei System) v33.0 技术架构手册

## 1. 概述
知微系统 v33.0 是一套高度自治、支持多模型动态调度、具备代码级自愈能力的 AI 自动化开发与运维体系。

## 2. 核心组件
### 2.1 编排器 (Orchestrator v33.0)
- **多模型协作**：使用 `qwen3-max` 进行高精度 DAG 任务拆解，配合 `MiniMax-M2.5` 进行指令优化。
- **动态演进**：集成 RAG 知识库，根据历史 [BugFix] 模式自动加固子任务 Prompt。
- **Token 效率方案 (v33.0)**：
    - **指针式指令**：编排器优先输出任务索引或搜索建议（如“Read RAG lessons”），将深度阅读负荷转嫁给后台 Worker。
    - **硬编码约束**：移除昂贵的二次 LLM 润色，通过 Python 模板直接注入工程安全约束。

### 2.2 动态路由器 (ModelRouter) [NEW]
- **智能分发**：根据任务复杂度实时分发资源。
- **模型矩阵**：
    - `qwen3-max`: 架构、规划、复杂复盘。
    - `qwen3-coder-plus`: 核心业务代码实现。
    - `glm-5` / `kimi-k2.5`: 长文本审计与大规模上下文分析。
    - `qwen3.5-plus`: 轻量级任务与流量承载。

### 2.3 自省代理 (CriticAgent v33.0)
- **多维打分**：基于 Logic、Safety、Quality (0-100) 深度复盘失败任务。
- **自动修复 (Self-Healing)**：对于低分任务，由 `qwen3-max` 生成修复建议并自动重新入列执行。
- **日志净化 (Denoising)**：在提交复盘前自动过滤控制码及冗长思考噪音，单次复盘 Token 消耗可降低 60% 以上。

### 2.4 消息总线 (MessageBus)
- **事件驱动**：废弃文件轮询，全体组件通过统一总线进行异步通信与飞书通知。

## 3. 安全与规范
- **脱敏机制**：所有 API 秘钥通过环境变量加载，禁止硬编码。
- **并发防护**：各仓库执行环境独立，具备 Git Lock 指数退避重试机制。

## 4. 关键文件索引
- `orchestrator.py`: 谋微编排引擎。
- `worker.py`: 开发执行 Worker。
- `model_router.py`: 智能调度路由。
- `critic_agent.py`: 自省与自愈代理。
- `task_store.py`: 任务持久化与依赖管理。
